Dns Master Slave Debian9

Erwin Kurniawan

October 7, 2018

Persiapan Server Debian

  • Dua VM server Debian satu network ( menggunakan HOST Only Adapter )
  • IP VM-Master : 192.168.0.3 /26
  • IP VM-Slave : 192.168.0.4 /26
  • Install bind9 dnsutils

Konfigurasi

VM-Master

1 Install bind9 & dnsutils

    apt-get install bind9 dnsutils -y

2 Jika sudah terinstall , sekarang masuk di directory /etc/bind

    cd /etc/bind

3 isikan konfigurasi di file named.conf.local dengan editor kesukaan anda seperti di bawah

    zone "erwin.org" {
        type master;
        allow-transfer { 192.168.0.4; };
        file "/etc/bind/erwin";
    };
    
    zone "0.168.192.in-addr.arpa" {
        type master;
        allow-transfer { 192.168.0.4; };
        file "/etc/bind/ip";
    };

4 kemudian buat file erwin & ip dari db.local dan db.127

    cp db.local erwin; cp db.127 ip

5 Edit file erwin dan isikan seperti di gambar

6 Edit file ip dan Isikan seperti di gambar

7 Kemudian restart servicebind9

    service bind9 restart

8 Selanjutnya tambahkan nameserver di /etc/resolv.conf

    echo -e "nameserver 192.168.0.3\nnameserver 192.168.0.4" > /etc/resolv.conf

9 Jika sudah cek dengan perintah nslookup pada kedua ip dari masing masing VM

    nslookup 192.168.0.3
    nslookup 192.168.0.4

10 Apabila sudah muncul domain yang di arahkan ke ip masing masing VM maka konfigurasi di VM-Master sudah berhasil

VM-Slave

1 Install bind9 & dnsutils

    apt-get install bind9 dnsutils -y

2 Jika sudah terinstall , sekarang masuk di directory /etc/bind

    cd /etc/bind

3 isikan konfigurasi di file named.conf.local dengan editor kesukaan anda seperti di bawah

    zone "erwin.org" {
        type slave;
        masters { 192.168.0.3; };
        file "erwin";
    };
    
    zone "0.168.192.in-addr.arpa" {
        type slave;
        masters { 192.168.0.3; };
        file "ip";
    };

4 Kemudian restart servicebind9

    service bind9 restart

5 Selanjutnya tambahkan nameserver di /etc/resolv.conf

    echo -e "nameserver 192.168.0.3\nnameserver 192.168.0.4" > /etc/resolv.conf

6 Jika sudah cek dengan perintah nslookup pada kedua ip dari masing masing VM

    nslookup 192.168.0.3
    nslookup 192.168.0.4

7 Apabila sudah muncul domain yang di arahkan ke ip masing masing VM maka konfigurasi di VM-Slave sudah berhasil

8 Selanjutnya cek di directory /var/cache/bind9

    ls /var/cache/bind9

9 apabila muncul file erwin & ip maka konfigurasi DNS Master Slave sudah Berhasil.
