Membuat virtualhost di debian 8.6

Erwin Kurniawan

November 1, 2017


A.Pendahuluan
  • Virtual hosting adalah metode untuk hosting beberapa nama domain (dengan penanganan terpisah dari setiap nama) pada satu server (atau kumpulan server). Hal ini memungkinkan satu server untuk berbagi sumber dayanya, seperti siklus memori dan prosesor, tanpa memerlukan semua layanan yang diberikan untuk menggunakan nama host yang sama. Istilah virtual hosting biasanya digunakan dalam referensi ke server web namun prinsipnya terbawa ke layanan internet lainnya.
B.Latar Belakang
  • Karena alasan penghematan sumber daya dari server maka alangkah baiknya kita menggunakan virtual host , ini sangat membantu dalam penghematan dari segi fisik server maupun aplikasi yang di install , karena ini hanya memerlukan 1 fisik , dengan DocumentRoot yang banyak
C.Maksud & Tujuan
  • Membuat Virtualhost
D.Hal Hal Yang Di Butuhkan
  • Server 
  • Apache2
  • Bind9
E.Langkah langkah
  1. Sebelum Membuat virtualhost ,pastikan sudah menginstall DNS server & Web server 
  2. Pertama tambahkan nama subdomain di bind , ini hanya di tambahkan di db.192 & db.roots 
  3. edit db.roots , dengan perintah nano /etc/bind/db.roots
  4. edit db.192 , dengan perintah nano /etc/bind/db.192
  5. kemudian masuk ke directory /etc/apache2/site-available , dengan perintah cd /etc/apache2/site-available/
  6. Selanjutnya copy 000-default.conf ke indra.conf , dengan perintah cp 000-default.conf indra.conf 
  7. Setelah di copy , edit indra.conf seperti di bawah
  8. kemudian enable-kan indra.conf , dengan perintah a2ensite indra.conf
  9. buat file index.html di /home/indra/public_html/ , dengan perintah nano /home/indra/public_html/index.html dan isikan seperti di bawah
  10. Setelah itu restart service dns dan webserver-nya
    • service bind9 restart
    • service apache2 restart
  11. Untuk hasil bisa di lihat di browser dengan subdomain yang telah di buat
